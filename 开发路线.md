# OfferAgent 开发路线

更新时间：2026-07-08

## 1. 产品定位

OfferAgent 的当前形态是“Offer 沙盘 + 面试官 Skill 库 + EvidenceGraph”的静态 Web MVP。它不是自动招聘决策系统，也不是黑盒打分器，而是面向候选人与面试官的面试准备、证据追踪和追问辅助工具。

核心目标：

1. 帮助候选人理解 JD、补齐项目证据、准备面试追问。
2. 帮助面试官快速识别风险、设计追问、记录人工反馈。
3. 用 EvidenceGraph 让 JD、简历、证据、问题、风险、反馈和 Offer 信号可以互相引用。
4. 用 OfferSimulationRun 把 Offer 推演从报告文本升级为可回填状态。
5. 用 SkillDefinition 为后续 Skill Registry 打基础。

## 3. 当前边界

1. 真实样本评测暂时不做。
2. 还没有用户账号、云端保存、团队协作。
3. 还没有 ATS / HRIS 集成。
4. Skill Registry 还不是完整运行时技能系统。
5. EvidenceGraph 还不是持久化知识图谱。
6. OfferSimulationRun 还没有跨会话保存。
7. 当前前端逻辑集中在 `apps/web/app.js`，后续需要模块化。

## 4. 下一步优先级

### P0：稳定当前产品体验

1. 清理剩余乱码和历史文案。
2. 压缩页面高度，继续减少无效留白。
3. 用 Playwright 对中文、英文、候选人、面试官、工作台、图谱做截图回归。
4. 把当前 smoke test 拆成更可维护的 UI / schema / report 三类检查。

### P1：让图谱成为决策工具

1. 强化节点跳转报告段落。
2. 增加图谱节点搜索。
3. 增加按风险等级、证据等级、Skill 来源过滤。
4. 展示反馈对节点和边的影响 diff。

### P2：把反馈闭环做实

1. 让人工反馈更结构化。
2. 保存反馈历史。
3. 将反馈变成 SkillDefinition 更新候选建议。
4. 支持规则冲突裁决。

### P3：模块化工程结构

建议拆分：

1. `i18n.js`：语言文案。
2. `roles.js`：RoleProfile。
3. `skills.js`：SkillDefinition 与选择逻辑。
4. `graph.js`：EvidenceGraph 构建与渲染。
5. `reports.js`：报告拼装。
6. `pdf.js`：PDF 导出。
7. `feedback.js`：FeedbackDistillation。
8. `offer.js`：OfferSimulationRun。

### P4：真实验证

当前暂不做真实样本评测。后续如果恢复，应优先验证：

1. 面试官是否愿意采用问题。
2. 报告是否能减少面试准备时间。
3. 证据图谱是否帮助定位风险。
4. 人工反馈是否能沉淀成可复用规则。
