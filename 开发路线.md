# OfferAgent 开发路线

更新时间：2026-07-09

## 1. 产品定位

OfferAgent 当前形态是“Offer 沙盘 + 面试官 Skill 库 + EvidenceGraph + 虚拟面试委员会 + 一致性模式”的静态 Web MVP。它不是自动招聘决策系统，也不是黑盒打分器，而是面向候选人与面试官的面试准备、证据追踪和追问辅助工具。

核心目标：

1. 帮助候选人理解 JD、补齐项目证据、准备面试追问。
2. 帮助面试官快速识别风险、设计追问、记录人工反馈。
3. 用 EvidenceGraph 让 JD、简历、证据、问题、风险、反馈、Offer 信号、Skill 和虚拟角色互相引用。
4. 用 OfferSimulationRun 把 Offer 推演从报告文本升级为可回填状态。
5. 用 SkillDefinition 和 VirtualPanel 为后续 Skill Registry 与多角色推演打基础。
6. 用一致性模式降低相同输入下报告漂移。

11. 图谱页下方以聊天气泡流式展示虚拟面试委员会讨论。

### 2.2 候选人与面试官模式

已完成：

1. 候选人和面试官保留同一套输入信息，切换视图不丢数据。
2. 候选人模式隐藏人工反馈。
3. 面试官模式显示人工反馈。
4. 候选人模式只显示候选人报告导出。
5. 面试官模式只显示面试官报告导出。

### 2.7 FeedbackDistillation

已完成：

1. 人工反馈写入报告。
2. 升级、降权、删除、保留规则展示。
3. 反馈影响说明。
4. Skill 更新建议。

### 2.8 SkillDefinition 与虚拟面试委员会

已完成 5 个 SkillDefinition 示例：

1. HR 面试官。
2. 业务负责人。
3. 项目推进面试官。
4. 谈薪顾问。
5. 决策层压力官。

已完成轻量 MiroFish 化结构：

1. `VirtualPanel`：从岗位、JD、简历和 Skill 生成虚拟面试官角色。
2. `PanelDiscussionRound`：生成多轮轻量讨论。
3. `ModeratorSummary`：汇总共识、分歧和最终建议。
4. EvidenceGraph 中可审计虚拟角色贡献。
5. 图谱页已支持虚拟委员会聊天气泡流式展示，并在生成完成后保留讨论历史。

### 2.9 一致性模式

已完成：

1. 输入指纹 `input_fingerprint`。
2. 结构化 JSON 中间层 `structured_evaluation`。
3. 本地缓存复用相同输入的基础报告。
4. 真实模型请求使用 `temperature: 0` 和 `seed`。
5. 缓存不保存 API Key，不保存人工反馈。
6. smoke test 增加一致性和隐私检查。

## 3. 当前边界

1. 真实样本评测暂时不做。
2. 还没有用户账号、云端保存、团队协作。
3. 还没有 ATS / HRIS 集成。
4. Skill Registry 还不是完整运行时技能系统。
5. EvidenceGraph 还不是持久化知识图谱。
6. OfferSimulationRun 还没有跨会话云端保存。
7. 虚拟面试委员会仍是轻量规则驱动，不是完整多 Agent 引擎。
8. 当前前端逻辑集中在 `apps/web/app.js`，后续需要模块化。

## 4. 下一步优先级

### P0：稳定当前产品体验

1. 清理剩余历史文案和编码问题。
2. 用 Playwright 做中文、英文、候选人、面试官、工作台、图谱截图回归。
3. 把当前 smoke test 拆成 UI / schema / report / cache 四类检查。
4. 增加缓存命中和未命中的浏览器级回归测试。

### P1：让虚拟面试委员会更可审计

1. 展示每个虚拟面试官的 stance、influence_weight、focus。
2. 展示每轮讨论提出的证据、质疑和建议。
3. 让主持人总结可以反向跳转到图谱节点和报告段落。
4. 支持按虚拟角色、讨论轮次、证据节点筛选聊天记录。

### P2：让图谱成为决策工具

1. 增加图谱节点搜索。
2. 增加按风险等级、证据等级、Skill 来源、虚拟角色来源过滤。
3. 展示反馈对节点和边的影响 diff。
4. 让虚拟角色的 challenges 直接提升问题优先级。

### P3：把反馈闭环做实

1. 让人工反馈更结构化。
2. 保存反馈历史。
3. 将反馈变成 SkillDefinition 更新候选建议。
4. 支持规则冲突裁决。
5. 让反馈影响虚拟角色权重。

### P4：模块化工程结构

建议拆分：

1. `i18n.js`：语言文案。
2. `roles.js`：RoleProfile。
3. `skills.js`：SkillDefinition 与选择逻辑。
4. `graph.js`：EvidenceGraph 构建与渲染。
5. `reports.js`：报告拼装。
6. `pdf.js`：PDF 导出。
7. `feedback.js`：FeedbackDistillation。
8. `offer.js`：OfferSimulationRun。
9. `consistency.js`：输入指纹、结构化中间层、缓存复用。
10. `virtual-panel.js`：VirtualPanel、PanelDiscussionRound、ModeratorSummary。

### P5：真实验证

当前暂不做真实样本评测。后续如恢复，应优先验证：

1. 面试官是否愿意采用问题。
2. 报告是否能减少面试准备时间。
3. 证据图谱是否帮助定位风险。
4. 人工反馈是否能沉淀成可复用规则。
5. 一致性模式是否显著降低同输入报告漂移。
