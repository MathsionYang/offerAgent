# OfferAgent 多视角评估

更新时间：2026-07-08

## 1. 综合结论

OfferAgent 当前已经从“路线设计”进入“可演示 Web MVP”阶段。核心闭环已经具备：输入简历与 JD、选择目标岗位、生成候选人 / 面试官报告、展示 EvidenceGraph、写入人工反馈、导出 PDF。

当前最有价值的方向不是继续堆报告段落，而是让证据图谱、反馈闭环、Offer 沙盘和 Skill 系统真正成为可追踪、可回填、可迭代的运行状态。

## 2. 产品视角

优势：

1. 定位清楚：不做黑盒打分，不替代人工录用决策。
2. 双角色清楚：候选人用于准备，面试官用于追问和反馈。
3. 交互更聚焦：工作台 / 图谱分离，顶部固定控件，报告生成后立即进入图谱。
4. 已覆盖四类岗位：产品经理、开发人员、技术支持人员、销售人员。
5. EvidenceGraph 让结论、证据、问题和反馈可以互相追溯。

风险：

1. 报告内容仍然偏多，首屏决策信息需要继续压缩。
2. 图谱目前是辅助展示，后续要继续强化为决策入口。
3. SkillDefinition 仍偏示例化，还不是成熟的 Skill Registry。
4. 真实样本验证暂时缺位，但这条当前明确暂不做。

## 3. 面试官视角

面试官真正关心的是：

1. 能否更快看出候选人与岗位的真实差距。
2. 能否给出可直接采用的追问。
3. 能否解释每个结论来自哪条 JD、简历证据或反馈。
4. 能否记录自己不同意系统判断的原因。

当前已满足：

1. 面试官模式显示人工反馈。
2. 面试官报告体现人工反馈。
3. 图谱节点可以跳转报告段落。
4. FeedbackDistillation 展示升级、降权、删除、保留。

下一步增强：

1. 反馈影响 diff。
2. 问题采用率统计。
3. 面试官自定义规则优先级。
4. Skill 更新建议的确认入口。

## 4. 候选人视角

候选人真正关心的是：

1. JD 到底在考什么。
2. 自己简历里哪些项目证据不足。
3. 哪些问题最可能被追问。
4. 如何准备更可信的项目叙述。

当前已满足：

1. 候选人模式隐藏人工反馈，减少干扰。
2. 候选人报告强调准备重点和追问题库。
3. 工作台中候选人简历与岗位 JD 左右布局，方便对照。
4. 可导出候选人报告。

下一步增强：

1. 候选人版报告再压缩首屏摘要。
2. 把“缺证点”转成准备清单。
3. 增加按岗位能力维度的准备进度。

## 5. 技术视角

优势：

1. 静态 Web MVP 易部署。
2. Mock Demo 与真实模型配置共存。
3. schema 已覆盖 EvaluationRun、OfferSimulationRun、EvidenceGraph、FeedbackDistillation。
4. smoke test 覆盖 UI 结构、schema、报告能力、导出逻辑。

风险：

1. `apps/web/app.js` 过大，后续维护成本会升高。
2. UI、报告生成、图谱、PDF、反馈、Offer 沙盘逻辑耦合较重。
3. 目前缺少浏览器级自动截图回归。

建议：

1. 下一阶段先模块化，不急于新增大功能。
2. 把 RoleProfile、SkillDefinition、EvidenceGraph、Report Builder 独立出来。
3. 增加 Playwright 截图回归，覆盖中文 / 英文、候选人 / 面试官、工作台 / 图谱。

## 6. 投资人 / 商业视角

优势：

1. 与普通简历分析工具不同，OfferAgent 强调证据链和人工反馈。
2. 双角色设计让候选人和面试官都能使用。
3. EvidenceGraph 与 FeedbackDistillation 有潜力形成长期数据资产。

风险：

1. 真实使用频率尚未验证。
2. 付费对象与采购路径尚未验证。
3. 数据合规、安全与企业信任体系尚未建设。

当前建议：

不急于包装成商业化招聘系统，先打磨 MVP，保持“面试准备 + 面试官追问辅助 + 证据链解释”的清晰定位。

## 7. 优先级建议

### P0：体验稳定

1. 修复剩余乱码。
2. 保持顶部控件一致。
3. 压缩页面高度和留白。
4. 增加截图回归。

### P1：图谱决策化

1. 节点搜索。
2. 风险等级过滤。
3. Skill 来源过滤。
4. 反馈影响 diff。

### P2：反馈闭环

1. 反馈历史。
2. 规则冲突处理。
3. Skill 更新建议确认。

### P3：工程模块化

1. 拆分 app.js。
2. 抽离 i18n。
3. 抽离 report / graph / feedback / offer / skill。

## 8. 当前不做

1. 真实样本评测。
2. ATS 集成。
3. 账号系统。
4. 团队协作。
5. 云端保存。
6. 完整 Skill 市场。
