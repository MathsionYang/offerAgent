# 统一语言投影设计

## 目标

使语言切换能够更新所有系统生成的输出界面，同时完整保留所有用户输入的文本原貌。

受影响的输出界面包括：

- 实时候选人报告、面试官报告和 Offer 报告。
- 证据图谱。
- 虚拟面试委员会。
- 决策摘要。
- 面试官评分表。
- PDF 内容、标题、元数据和文件名。

## 产品不变式

1. 简历、JD、公司背景、Offer 约束条件、目标等级文本、API 配置及其他用户输入的值，永远不被翻译或覆盖。
2. 切换语言不会重新生成评估结果、不会改变证据分类、不会改变决策、不会改变输入指纹。
3. 报告和图谱详情中引用的原文摘录保持其原始语言不变。
4. 系统标签、分析内容、推荐建议、提问、风险描述、委员发言、反馈摘要以及导出界面元素跟随当前选择的语言。
5. 切换回此前使用过的语言时，复用已缓存的本地化产物。

## 架构

### 规范运行（Canonical Run）

`currentRun` 仍然是规范的评估状态。现有的结构化字段和 `input_snapshot` 保持为唯一事实来源。

规范运行新增以下字段：

```js
localized_artifacts: {
  zh: {
    schema_version: "language-artifact.v1",
    source: "generated" | "translated" | "projected",
    report_markdown: "...",
    text_by_id: {
      "question:q_1": "...",
      "risk:risk_1": "...",
      "panel:round_1:agent_hr": "...",
      "moderator:final_recommendation": "..."
    }
  },
  en: { ... }
}
```

产物缓存仅包含系统生成的文本，不得包含 API 密钥或原始用户输入字段的翻译副本。

### 语言投影模块

新增 `apps/web/src/localized-run-view.js`。

该模块对外暴露以下接口：

- `projectRunForLanguage(run, language)` —— 按语言投影运行
- `collectTranslatableArtifacts(run)` —— 收集可翻译产物
- `mergeLocalizedArtifacts(run, language, artifact)` —— 合并本地化产物
- `isSourceExcerptField(fieldName)` —— 判断是否为原文摘录字段
- `resolveLocalizedText(run, language, stableId, fallback)` —— 解析本地化文本

`projectRunForLanguage` 返回一个仅用于展示的克隆对象。它通过稳定 ID 本地化枚举值并替换系统生成的文本，同时保持证据源字段不变。

所有输出模块接收投影后的运行对象，而非直接修改或渲染规范运行对象。

## 翻译策略

### Mock 与确定性输出

Mock 运行使用现有的中文和英文构建器。应用在本地构建并缓存目标语言的报告，无需发起外部请求。

结构化枚举和已知模板文本通过现有的本地化映射器进行本地化。

### 真实模型输出

当目标语言的产物缺失时：

1. 仅将系统生成的文本收集为稳定 ID 的 JSON 载荷。
2. 通过现有模型客户端发送一个非流式本地化请求。
3. 指示模型：
   - 仅返回 JSON。
   - 保留每一个稳定 ID。
   - 保留数字、URL、标识符、Markdown 结构和产品名称。
   - 绝不翻译引用的简历、JD、公司背景、目标等级或 Offer 约束条件的摘录。
4. 验证响应结构。
5. 将有效的翻译条目合并到 `localized_artifacts[targetLanguage]` 中。
6. 持久化运行缓存。

如果翻译失败，静态 UI 和确定性枚举本地化仍然会切换。报告保持其源语言产物，状态区域显示本地化失败消息。不修改任何规范评估数据。

## 语言切换流程

`applyLanguage` 变为异步操作。

1. 设置 `currentLanguage` 并立即更新静态 UI 控件。
2. 如果当前没有运行对象，正常结束。
3. 禁用语言选择器并显示本地化进度状态。
4. 解析或创建目标语言的产物。
5. 为所选语言构建投影运行对象。
6. 重新渲染：
   - 报告预览。
   - 决策摘要。
   - 评分表。
   - 证据图谱。
   - 虚拟面试委员会。
7. 重新启用语言选择器。

使用单调递增的请求令牌，防止较早的慢速翻译覆盖较新的语言选择结果。

## 模块改动

### 报告构建器

- 从当前显示语言选择构建器，而非 `input_snapshot.language`。
- 使用本地化的报告产物提取生成的内容段落。
- 引用证据时继续使用原始 input snapshot 字段。

### 证据图谱

- 通过投影后的节点 ID 本地化节点标签和摘要。
- 保留 `source_excerpt`、简历证据、JD 证据及其他用户来源元数据。
- 语言切换后重新应用当前的搜索和筛选状态。

### 虚拟面试委员会

- 本地化角色名称、关注点、立场、阶段、影响、主张和主持人总结。
- 保留追踪 ID 和证据引用。

### 决策摘要与评分表

- 本地化生成的问题、推荐建议、反馈摘要和状态文本。
- 保留简历和 JD 证据摘录。

### PDF 导出

- 导出 `currentLanguage` 对应的投影运行对象。
- 使用当前语言的标题、元数据标签、日期区域设置和文件名。
- 不使用 `input_snapshot.language` 作为导出语言。

### 缓存

- 输入指纹保持不变。
- `localized_artifacts` 可随运行对象一起持久化。
- 运行时凭据仍然被排除。

## 错误处理

- 无效的本地化 JSON 会被拒绝，不会部分修改运行对象。
- 缺失翻译 ID 时回退到原始系统文本，并报告为本地化不完整。
- 合并前检查原文摘录，确保翻译载荷不能覆盖它们。
- 语言选择器在 `finally` 块中重新启用。

## 测试

### 单元测试

- 投影逐字节保留每个用户输入字段。
- 投影本地化系统枚举和稳定 ID 文本。
- 产物缺失时安全回退。
- 产物合并不接受源输入字段和未知结构。
- 报告构建器跟随显示语言而非生成语言。
- 图谱节点保留原文摘录的同时本地化生成的摘要。
- 委员消息和评分表问题使用本地化产物值。
- PDF 模板使用当前显示语言。
- 翻译载荷排除用户输入字段。
- 重复语言切换复用缓存产物。

### 浏览器回归测试

双向测试：

- 用中文生成，切换到英文，再切换回中文。
- 用英文生成，切换到中文，再切换回英文。

每个方向验证：

- 输入字段未改变。
- 报告、图谱、委员会、摘要、评分表和导出控件切换了语言。
- 重新渲染后图谱搜索仍然正常工作。
- 没有残留的混合语言系统标题。
- 没有控制台错误或水平溢出。

## 非目标

- 翻译用户输入的内容。
- 语言变化时重新评估候选人。
- 改变评估 Schema 或决策规则。
- 超出现有错误包装器，翻译第三方提供商直接返回的 API 错误。
