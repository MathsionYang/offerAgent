# OfferAgent 一页决策摘要标准

这份标准用于约束候选人报告和面试官报告的首页信息密度。目标是让 HR、招聘经理或业务面试官在 3 分钟内看懂：这个候选人是否值得推进、为什么、下一轮必须问什么。

## 1. 设计目标

- 首页不是完整报告，而是决策入口。
- 首页必须先给结论，再给证据和行动。
- 首页只展示最重要的内容，细节下沉到证据链、问题库和 Offer 推演。
- 首页不输出自动录用或淘汰结论，只输出辅助判断和待验证问题。

## 2. 必含模块

| 模块 | 内容 | 长度约束 |
| --- | --- | --- |
| 闸口结论 | 匹配进入、条件性进入、不匹配不推进 | 1 行 |
| 最大亮点 | 最能支撑 JD 的项目证据 | 1-2 行 |
| 最大风险 | 最影响推进的缺证项或低可信证据 | 1-2 行 |
| 必问问题 | 下一轮最该问的 3-5 个问题 | 每题 1 行 |
| 证据可信度 | 一级 / 二级 / 三级证据数量 | 1 行 |
| 下一步动作 | 推进、补材料、加面、暂缓或谈薪前验证 | 1-2 行 |

## 3. 候选人版首页

候选人版首页应回答：

- 我现在最大的优势是什么？
- 我最容易被追问或质疑什么？
- 今晚最应该补哪 3 件事？
- 如果我是转岗 / 跨行业，应该怎么解释迁移能力？

建议结构：

```text
三秒结论：
最大优势：
最大风险：
今晚行动清单：
面试中主动引导：
需要避免的说法：
```

## 4. 面试官版首页

面试官版首页应回答：

- 是否建议进入下一轮？
- 哪个项目最值得追问？
- 哪个能力缺证最严重？
- 哪 3-5 个问题最能拉开差距？
- 面试后应该把什么反馈回填？

建议结构：

```text
一分钟决策：
推荐等级：
关键证据：
关键风险：
必问追问：
面试动作：
给下一轮面试官的话：
```

## 5. Offer 版首页

Offer 推演首页应回答：

- 当前是否具备进入 Offer 沙盘的基础？
- 哪些证据可以支撑职级或谈薪？
- 哪些证据还不能转化为 Offer 溢价？
- 谈薪前必须确认什么？

建议结构：

```text
沙盘结论：
Offer 杠杆：
最大推进风险：
谈薪前验证：
面试后回填：
```

## 6. 内容下沉规则

首页不展开：

- 完整证据链。
- 全部问题库。
- 多角色视角详情。
- 七步 Offer 推理长表。
- Prompt / 模型过程解释。
- 大段候选人画像。

这些内容应下沉到后续章节、结构化运行状态或 EvidenceGraph 详情中。

## 7. 红线

首页不得出现：

- 自动录用、自动淘汰、必须拒绝等最终决策措辞。
- 基于年龄、性别、婚育、照片、籍贯等敏感字段的判断。
- 没有证据支持的强结论。
- 过度确定的潜力预测。
- 让候选人感到被审讯的攻击性措辞。

## 8. 验收标准

一页摘要通过的标准：

- 3 分钟内能读完。
- 读完能复述闸口结论、最大亮点、最大风险。
- 至少 3 个问题可直接进入面试。
- 每个关键结论都有证据入口。
- 不需要理解系统方法论也能使用。
