# OfferAgent MVP 评测表模板

这份模板用于验证当前 Web MVP 是否真正能帮助 HR、招聘经理和业务面试官完成“面试前辅助准备”。评测目标不是证明功能多，而是判断系统生成的闸口结论、证据链、追问问题和报告摘要是否可被真实招聘参与者采用。

## 1. 使用场景

- 适用阶段：产品经理、开发人员、技术支持人员、销售人员等目标岗位的业务面试前准备。
- 样本规模：第一轮至少 10 份脱敏简历 + JD。
- 评审人员：至少 3 位招聘参与者，建议覆盖 HR、招聘经理、业务面试官。
- 输出形式：每个样本保留当前运行内结构化 EvaluationRun 状态、EvidenceGraph、OfferSimulationRun、一份 PDF 报告和一行人工评测记录；不再要求用户侧导出评测 JSON。

## 2. 样本信息表

| 字段 | 填写说明 |
| --- | --- |
| sample_id | 样本编号，例如 PM-001 |
| role_type | 岗位类型，例如 product_manager、developer、technical_support、sales |
| jd_source | JD 来源，例如真实脱敏、模拟 JD、公开 JD |
| resume_source | 简历来源，例如真实脱敏、模拟简历、历史样本 |
| candidate_level | 候选人目标职级或经验段 |
| evaluator | 评审人角色，例如 HR、业务面试官、招聘经理 |
| run_id | Web MVP 生成的 EvaluationRun id |
| model_mode | mock 或 llm |
| model_name | 模型名称，Mock 模式填写 mock-product-manager-v1 |

## 3. 核心评分表

| 评估项 | 评分口径 | 分值 |
| --- | --- | ---: |
| 硬性匹配一致性 | 系统闸口与人工判断是否一致 | 1-5 |
| 证据可信度 | 关键结论是否能追溯到简历 / JD / 上下文 | 1-5 |
| 追问可采用性 | 面试官是否愿意直接采用或改写采用问题 | 1-5 |
| 报告可读性 | 3 分钟内是否能抓住亮点、风险和下一步 | 1-5 |
| 风险识别价值 | 是否发现人工容易忽略的风险点 | 1-5 |
| 候选人体验风险 | 问题是否专业、具体、不过度压迫 | 1-5 |
| Offer 推演参考价值 | 谈薪、到岗、职级和推进建议是否有参考 | 1-5 |

建议通过标准：

- 硬性匹配一致性：10 份样本中至少 7 份基本一致。
- 追问可采用性：系统生成的问题超过 50% 被采用或改写采用。
- 报告可读性：多数评审人能在 3 分钟内说出最大亮点、最大风险和下一轮追问。
- 证据可信度：关键结论不能出现明显证据误引。

## 4. 问题采用记录

| 字段 | 填写说明 |
| --- | --- |
| generated_question_count | 系统生成的候选问题数量 |
| reviewed_question_count | 面试官实际看过的问题数量 |
| adopted_question_count | 直接采用的问题数量 |
| rewritten_question_count | 改写后采用的问题数量 |
| rejected_question_count | 明确不采用的问题数量 |
| adoption_rate | `(adopted + rewritten) / reviewed` |
| best_question | 最有价值的一个问题 |
| low_value_question | 最低价值或最泛化的问题 |

采用率口径建议使用“面试官实际看过的问题”作为分母，而不是系统生成的全部问题。这样更贴近真实工作流。

## 5. 误判记录

| 字段 | 填写说明 |
| --- | --- |
| system_gate | 系统闸口：匹配进入、条件性进入、不匹配不推进 |
| human_gate | 人工判断：推进、谨慎推进、暂缓、不推进 |
| is_consistent | 是否基本一致 |
| disagreement_reason | 证据不足、权重不同、岗位理解不同、系统误判 |
| affected_capability | 受影响的能力维度 |
| evidence_issue | 证据误引、缺证、低估、过度推断、表达过长 |
| recommended_fix | 建议修正动作 |

## 6. 面试后回填

| 字段 | 填写说明 |
| --- | --- |
| asked_questions | 面试中实际问到的问题 |
| confirmed_evidence | 被候选人证实的证据 |
| disproved_evidence | 被候选人推翻的证据 |
| new_risk | 面试中新暴露的风险 |
| updated_gate | 面试后的推进判断 |
| next_interviewer_note | 给下一轮面试官的一句话 |

## 7. 汇总看板

第一轮 10 份样本跑完后，至少汇总以下指标：

- 硬性匹配一致率。
- 系统问题采用率。
- 平均报告可读性评分。
- 平均证据可信度评分。
- 高频误判原因 Top 3。
- 高频缺证能力维度 Top 3。
- 面试官最常采用的问题类型。
- 需要下线、重写或降权的问题类型。

每轮评测结束后，用以下格式输出复盘：

```text
最有效的能力：
最薄弱的能力：
最常见误判：
优先修正项：
是否进入下一阶段：
```
